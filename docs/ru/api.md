# Руководство по Gateway API

В этом руководстве описаны методы PHP-биндингов. Главный источник сведений о
поведении сервера и ценах —
[документация Telegram Gateway API](https://core.telegram.org/gateway/api).

## Отправка сообщения с кодом

Вызов:

```php
$status = $api->sendVerificationMessage(
    phoneNumber: '+12025550123',
    codeLength: 6,
    ttl: 60,
);
```

Аргументы метода:

| Аргумент | Обязательный | Правило |
| --- | --- | --- |
| `phoneNumber` | Да | Используйте формат E.164. |
| `requestId` | Нет | Передайте идентификатор из успешного `checkSendAbility`. Первая отправка с этим идентификатором не приводит к повторному списанию. |
| `senderUsername` | Нет | Укажите проверенный канал, который принадлежит владельцу токена. |
| `code` | Нет | Используйте от 4 до 8 цифр. Этот аргумент имеет приоритет над `codeLength`. |
| `codeLength` | Нет | Укажите число от 4 до 8, если код должен создать Telegram. |
| `callbackUrl` | Нет | Укажите HTTPS-адрес для отчётов о доставке. |
| `payload` | Нет | Используйте поле для внутренних данных. Telegram не показывает его пользователю. |
| `ttl` | Нет | Укажите от 30 до 3600 секунд. |

Метод возвращает `RequestStatusInterface`.

## Проверка возможности отправки

Вызов:

```php
$status = $api->checkSendAbility('+12025550123');
```

Метод необязателен. Не используйте его как бесплатную пробную проверку. Если
Telegram подтвердит возможность отправки, со счёта могут быть списаны средства.

Успешный ответ содержит `requestId`. Передайте его в один последующий вызов
`sendVerificationMessage`. Повторного списания не будет. Ещё одна отправка с тем
же идентификатором завершится ошибкой.

## Проверка статуса и кода

Получите текущий статус:

```php
$status = $api->checkVerificationStatus($requestId);
```

Попросите Telegram проверить введённый пользователем код:

```php
$status = $api->checkVerificationStatus($requestId, $code);

if ($status->verificationStatus?->status === 'code_valid') {
    // Продолжите вход пользователя в приложение.
}
```

Вызывайте этот метод после ввода кода пользователем. Telegram рекомендует
делать это и в том случае, если код создало ваше приложение.

## Отзыв сообщения

Вызов:

```php
$accepted = $api->revokeVerificationMessage($requestId);
```

Значение `true` означает, что Telegram принял запрос. Оно не подтверждает
удаление сообщения. Telegram не удаляет уже доставленное или прочитанное
сообщение.

## Статус запроса

| Свойство | Тип | Значение |
| --- | --- | --- |
| `requestId` | `string` | Идентификатор запроса. |
| `phoneNumber` | `string` | Номер получателя в формате E.164. |
| `requestCost` | `float` | Стоимость, которую создала проверка или отправка. |
| `isRefunded` | `?bool` | Признак возврата средств. |
| `remainingBalance` | `?float` | Остаток после операции со списанием. |
| `deliveryStatus` | `?DeliveryStatusInterface` | Статус доставки отправленного сообщения. |
| `verificationStatus` | `?VerificationStatusInterface` | Статус проверки кода. |
| `payload` | `?string` | Внутренние данные приложения. |

Telegram может не вернуть любое необязательное поле. В PHP его значение будет
равно `null`.

## Статусы доставки

- `sent`
- `delivered`
- `read`
- `expired`
- `revoked`

Telegram может добавить новое значение. Безопасно обрабатывайте неизвестные
строки.

## Статусы проверки

- `code_valid`
- `code_invalid`
- `code_max_attempts_exceeded`
- `expired`

Telegram может добавить новое значение. Безопасно обрабатывайте неизвестные
строки.

## Списания и возвраты

Правила тарификации могут измениться. Проверяйте официальную документацию перед
выпуском приложения.

Правила, которые влияют на логику приложения:

1. Успешный вызов `checkSendAbility` может привести к списанию.
2. Одна отправка с полученным `request_id` не приводит к повторному списанию.
3. Отправка без этого идентификатора создаёт другой запрос.
4. Если сообщение не выполнило условия доставки за время `ttl`, возможен
   автоматический возврат.
5. Отзыв сообщения не приводит к возврату.

Не выполняйте реальную проверку в автоматическом тесте. Используйте локальный
ответ, как в [`examples/offline.php`](../../examples/offline.php).
