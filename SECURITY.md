# Security policy

[English](#english) · [Русский](#русский)

## English

### Report a vulnerability

Do not create a public issue for a suspected vulnerability.

Use a
[private GitHub security advisory](https://github.com/phenogram/gateway-bindings/security/advisories/new).
Include the affected version, the impact, and the minimum steps to reproduce the
problem.

Remove all access tokens, phone numbers, and verification codes from the report.
We will confirm receipt and assess the report before we publish details.

### Supported versions

Security fixes target the latest stable release and the default branch. A fix
can require an upgrade to a new stable release.

## Русский

### Сообщение об уязвимости

Не создавайте открытую задачу для предполагаемой уязвимости.

Используйте
[закрытое уведомление GitHub](https://github.com/phenogram/gateway-bindings/security/advisories/new).
Укажите затронутую версию, возможные последствия и минимальные шаги для
воспроизведения.

Удалите из отчёта токены доступа, номера телефонов и коды подтверждения. Мы
подтвердим получение и проверим отчёт до публикации сведений.

### Поддерживаемые версии

Исправления безопасности предназначены для последней стабильной версии и ветки
по умолчанию. Для получения исправления может потребоваться обновление до новой
стабильной версии.
